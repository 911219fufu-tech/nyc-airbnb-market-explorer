# Presentation Slides Plan

## Recommended Length

8 to 12 slides

Recommended target:

- 10 slides

## Proposed Slide Flow

### 1. Title

- project title
- name
- course

### 2. Problem and Motivation

- what challenge exists
- why this dataset/problem matters

### 3. Target Users and Tasks

- who uses the system
- what they want to learn

### 4. Dataset

- source
- size
- variables
- limitations

### 5. Design Approach

- why this visualization system structure was chosen
- key principles from the course

### 6. System Overview

- dashboard layout
- main views
- interaction flow

### 7. Key Visualization 1

- show one major view
- explain what it reveals

### 8. Key Visualization 2

- show another major view
- explain linked interaction or comparison behavior

### 9. Insights

- 3 concise findings

### 10. Contributions and Future Work

- what the project contributes
- what can be improved next

## Slide Quality Rules

- keep text short
- use large readable visuals
- one message per slide
- prefer highlighted takeaways over dense paragraphs
- use screenshots with callouts when needed

## Demo Strategy

If a live demo is included:

- start from default dashboard state
- show one filtering example
- show one linked-view interaction
- end with a concrete insight
